PHASE 2 — MEETING COACH & DONNEES (Mai, ~4 semaines)
================================================================================
La feature la plus impactante, plus le branchement de donnees manquantes.

[ ] 5. Page "Meeting Coach" (~2 semaines)
    LE differenciateur de SalesOS.
    Architecture :
    - Input : lien Claap (ou transcript colle) + type de meeting
      -> Types : disco, follow-up, closing, QBR
    - Claude analyse avec une grille adaptee au type :
      -> MEDDIC pour la disco
      -> SPIN pour le follow-up
      -> Challenger pour le closing
    - Output structure :
      -> Resume du meeting
      -> Ce qui a bien marche
      -> Ce qui manque (infos non collectees, objections non traitees)
      -> Score par critere methodologique
      -> Conseils pour le prochain meeting
    - Historique des debriefs par deal
    - API route : /api/meeting-coach/analyze
    - Page : app/meeting-coach/page.tsx
    - Si Claap a une API, integrer la recup automatique du transcript

[ ] 6. Pluger les donnees recentes (~1 semaine)
    - Connecter les donnees provider/coaching au bot et aux briefings
    - Definir avec l'equipe les sources exactes
      (Notion ? Google Sheets ? API interne ?)
    - Creer un outil IA read_coaching_data dans le chat
    - S'assurer que les briefings utilisent des donnees fraiches

[ ] 7. Rapport hebdo automatique Slack (~2-3 jours)
    - Chaque lundi matin, message Slack auto dans le canal sales :
      -> Deals a risque (sante rouge/orange)
      -> Meetings de la semaine
      -> Signaux concurrentiels importants
      -> Top actions recommandees
    - Toutes les donnees existent deja, c'est un cron qui agrege + envoie
    - Peu d'effort, cree un rendez-vous recurrent avec l'outil

AMELIORER LA PARTIE COMPETITION (a reflechir plus amplement)
================================================================================
Le module competitive existe deja (battlecards, signaux, rapports, chat IA,
analyse batch via Tavily). Voici ce qui manquerait pour en faire un vrai
avantage concurrentiel.

[ ] 11. Lien competition <-> deals
    - Quand un concurrent est detecte sur un deal, l'afficher dans le
      scoring ET dans la fiche deal
    - Historique : sur quels deals on a croise tel concurrent ?
    - Win rate par concurrent (on gagne combien de fois face a X ?)
    - Quand on perd, pourquoi ? (lien avec Win/Loss analysis)

[ ] 12. Alertes concurrentielles en temps reel
    - Notification Slack/in-app quand un nouveau signal est detecte
      (levee de fonds, lancement produit, recrutement massif)
    - Seuil de confiance configurable (ne pas spammer)
    - Digest quotidien vs alerte immediate selon la gravite

[ ] 13. Trap questions / Landmines pour les commerciaux
    - Pour chaque concurrent, generer des questions "pieges" a poser
      en meeting pour exposer les faiblesses du concurrent
    - Ex : "Comment gerent-ils le multi-langue ?" si c'est un point faible
    - Integrer dans les battlecards existantes
    - Adapter les questions au type de meeting (disco, demo, closing)

[ ] 14. Matrice de positionnement dynamique
    - Tableau comparatif features/pricing auto-genere
    - Axes configurables (prix, maturite, vertical, taille cible)
    - Mis a jour automatiquement quand de nouveaux signaux arrivent
    - Exportable en PDF/image pour les decks commerciaux

[ ] 15. Integration competition <-> Meeting Coach
    - Quand un concurrent est mentionne dans un transcript,
      le detecter automatiquement
    - Analyser comment le commercial a reagi (a-t-il utilise
      les arguments de la battlecard ? a-t-il pose les trap questions ?)
    - Conseils post-meeting specifiques a la situation concurrentielle

[ ] 16. Veille competitive automatisee enrichie
    - Ajouter des sources au-dela de Tavily :
      -> G2/Capterra (avis clients concurrents)
      -> LinkedIn (recrutements, posts des dirigeants)
      -> Crunchbase (funding, acquisitions)
      -> ProductHunt (lancements)
    - Timeline visuelle de l'activite de chaque concurrent
    - Tendances sur 3/6/12 mois (le concurrent accelere ou ralentit ?)

[ ] 17. Objection handling training
    - Simulateur de conversation avec Claude qui joue le role
      d'un prospect qui cite un concurrent
    - Le commercial s'entraine a repondre aux objections
    - Score et feedback sur la qualite des reponses
    - Base sur les vraies objections rencontrees (historique deals)

[ ] 18. Competitive intelligence partagee
    - Quand un commercial apprend quelque chose sur un concurrent
      en meeting, il peut le remonter en 1 clic
    - Feed interne "intel terrain" visible par toute l'equipe
    - Claude synthetise les remontees terrain + signaux web
      pour mettre a jour les battlecards automatiquement
